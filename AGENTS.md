- Keep project insights in [INSIGHTS.md](/Users/aeron/neuroscience/INSIGHTS.md). This is the only progress and insight log unless the user explicitly asks for another artifact.
- Add a new entry for every meaningful research finding, scope decision, design decision, implementation decision, or unresolved concern.

- Add new entries at the top of the log, directly below the introductory sentence, so the file remains reverse chronological.
- Keep entries concise but substantive: record the decision or insight, why it matters, and any immediate implication for the MVP.

- Before finishing a work session, check whether the latest meaningful progress has been recorded in `INSIGHTS.md`.

## Project Direction

- Use `behavtaskatlas` as the working project name.
- Treat the project as an open-science atlas for behavioral neuroscience tasks, not merely a bibliography.
- Prefer machine-readable schemas, reusable trial-level data formats, explicit provenance, and links to existing archives over ad hoc prose-only curation.
- Preserve distinctions between operational task variables and cognitive interpretations.
- Keep the MVP focused on sensory-guided decision-making tasks unless the user explicitly broadens the scope.
