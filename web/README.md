# behavtaskatlas web

Astro 5 + Tailwind v4 + TypeScript app that consumes the JSON exports
generated by the parent Python project (`derived/manifest.json`,
`derived/catalog.json`, etc.) and renders the public catalog.

## Local dev

```bash
# from the repo root, regenerate the JSON inputs first:
uv run behavtaskatlas site-index

# then in this directory:
bun install
bun run dev    # http://127.0.0.1:4321
```

## Build

```bash
bun run build  # writes ./dist
```

## Deploy

Vercel is configured via the repo-root `vercel.json`. The build runs
`scripts/vercel-build.sh`, which installs `uv` and `bun`, regenerates the
static JSON exports from the YAML records, and then builds Astro.

## Architecture

- `src/lib/data.ts` — single import point for `manifest.json` and
  `catalog.json`; all other pages consume data through this module.
- `src/pages/index.astro` — overview, health dashboard, comparison table.
- `src/pages/catalog.astro` — task family / protocol / dataset / slice
  tables.
- `src/pages/protocols/[id].astro` — dynamic detail pages from
  `catalog.protocol_details`.
- `src/pages/datasets/[id].astro` — dynamic detail pages from
  `catalog.dataset_details`.
- Tailwind v4 is loaded via `@tailwindcss/vite`; theme tokens live in
  `src/styles/global.css`.

The Python static-site generator (`src/behavtaskatlas/static_site.py`)
remains the schema authority and is not yet retired — the Astro app is
running side-by-side until parity (Phase 3 of the migration plan).
